# 데이터베이스 연결 — 처음 따라하기 (Windows)

이 프로젝트는 **PostgreSQL** 에 데이터를 저장합니다.

> **추천:** 예전처럼 **클라우드 DB 주소만 넣고 로컬에서 개발**하려면  
> **`따라하기-클라우드DB-Neon.md`** 를 먼저 보세요. (Docker 불필요)  
> 아래는 **내 PC에 Docker로 PostgreSQL 올리는 방법**입니다.

---

## 0단계 — 지금 컴퓨터 상태 (확인됨)

- **Node.js** : 설치되어 있음 (개발용 도구)
- **Docker** : 아직 없음 → 1단계에서 설치

---

## 1단계 — Docker Desktop 설치 (가장 쉬운 방법)

PostgreSQL을 직접 설치하는 것보다, 이 프로젝트에 맞춰 둔 **도커**로 한 번에 올리는 방식이 편합니다.

1. 브라우저에서 **Docker Desktop for Windows** 를 검색하거나 아래 공식 페이지로 이동합니다.  
   https://www.docker.com/products/docker-desktop/
2. **Download for Windows** 를 눌러 설치 파일을 받습니다.
3. 받은 파일을 실행해 설치합니다. (재시작을 요청하면 재시작)
4. 설치 후 **Docker Desktop** 을 실행합니다.  
   - 트레이(시계 옆)에 고래 아이콘이 있고, “Docker is running” 비슷한 상태면 준비 완료입니다.
5. **WSL 2** 설치 안내가 나오면, 안내에 따라 **WSL 2** 를 켜두는 것이 좋습니다. (Docker 설치 화면이 알려 줍니다.)

> Docker 없이 가고 싶다면 나중에 “PostgreSQL 직접 설치”로도 가능하지만, 단계가 더 많습니다. 우선 Docker 방식을 권장합니다.

---

## 2단계 — 프로젝트에서 데이터베이스 컨테이너 켜기

1. **파일 탐색기**에서 이 폴더를 엽니다.  
   `D:\seongjang`
2. 폴더 안에서 **주소창**을 한 번 클릭하고, `cmd` 라고 입력한 뒤 **Enter** 를 누릅니다.  
   → 검은 창(명령 프롬프트)이 열립니다.
3. 아래 줄을 **통째로 복사**해서 창에 붙여넣고 **Enter** 합니다.

```text
docker compose up -d
```

4. 잠시 기다린 뒤, 오류 없이 끝나면 성공입니다.  
   - `docker` 를 찾을 수 없다고 나오면 → 1단계에서 Docker Desktop이 **실행 중**인지 확인하세요.

---

## 3단계 — 설정 파일 `.env` 만들기

1. 파일 탐색기에서 이 폴더를 엽니다.  
   `D:\seongjang\web`
2. 그 안에 **`.env.example`** 이라는 파일이 있습니다.
3. 그 파일을 **복사**해서 같은 폴더에 붙여넣기 합니다.
4. 복사본 이름을 **`.env`** 로 바꿉니다.  
   - Windows에서 “`.env.example`의 복사본”이 되면, 이름을 `.env` 로 수정하세요.  
   - “확장자를 바꾸시겠습니까?” 나오면 **예** (이름이 정확히 `.env` 여야 합니다.)
5. `.env` 파일을 메모장으로 열어서, 아래 한 줄이 있는지 확인합니다.

```text
DATABASE_URL="postgresql://seongjang:seongjang_dev@localhost:5432/seongjang"
```

2단계를 그대로 따라했다면 **수정 없이** 두어도 됩니다.

6. (권장) 같은 파일에서 `AUTH_SECRET=""` 부분에, 아무 긴 랜덤 문자열을 넣습니다.  
   - 비워도 당장 DB 연결 테스트에는 문제 없을 수 있지만, 로그인 쓸 때는 채우는 것이 좋습니다.

---

## 4단계 — DB 테이블 만들기 (마이그레이션)

1. 다시 명령 프롬프트를 엽니다 (2단계와 같이 `D:\seongjang` 에서 `cmd` ).
2. 아래를 **한 줄씩** 복사해서 실행합니다.

```text
cd web
```

```text
npx prisma migrate deploy
```

3. 끝에 **성공** 비슷한 메시지가 나오고, **에러**가 없으면 됩니다.  
   - `Can't reach database server` 가 나오면 → Docker Desktop 켜짐 + 2단계 `docker compose up -d` 다시 확인.

---

## 5단계 — 처음 데이터 넣기 (시드)

같은 창에서 (`D:\seongjang\web` 에 있는 상태):

```text
npm run db:seed
```

끝에 `Seed OK` 비슷한 말이 나오면 성공입니다.  
테스트 계정(개발용):

- 이메일: `dev@seongjang.local`
- 비밀번호: `devpassword`

---

## 6단계 — 웹 앱 실행해 보기

같은 폴더(`D:\seongjang\web`)에서:

```text
npm run dev
```

브라우저에서 `http://localhost:3000` 을 엽니다.

---

## 내가 이미 프로젝트에 해둔 것 (당신이 할 필요 없음)

- DB 테이블 정의와 **첫 마이그레이션 파일**은 저장소에 넣어 두었습니다.
- `npm run db:up` 은 `web` 폴더에서 Docker를 켤 때 쓸 수 있는 단축 명령입니다.  
  (터미널에서 `cd D:\seongjang\web` 후 `npm run db:up` 도 같은 효과입니다.)

---

## 자주 나는 말(에러)과 대처

| 메시지 / 상황 | 할 일 |
|----------------|--------|
| `docker`를 찾을 수 없습니다 | Docker Desktop 설치 후 **실행** |
| DB에 연결할 수 없습니다 | Docker 켜기 → `D:\seongjang`에서 `docker compose up -d` 다시 |
| `.env` 가 없다고 함 | 3단계대로 `web\.env` 파일 만들기 |

---

여기까지 하면 **데이터베이스 연결 준비는 끝**입니다.  
어느 단계에서 멈췄는지 **번호**만 알려주시면, 그 단계만 더 풀어서 적어 드리겠습니다.
