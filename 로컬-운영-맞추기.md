# 로컬 개발 ↔ 프로덕션(Vercel) 같게 쓰기 · 문제 확인

## 1. 생년월일 UI가 로컬과 프로덕션에서 다르게 보이던 이유

- 예전에는 `<input type="date">`(브라우저 기본 달력)만 썼습니다.
- **Chrome 데스크톱 / 모바일 Safari / 기기 OS 설정**에 따라 열리는 달력의 **년·월·일 순서·모양**이 달라집니다.
- **서버(로컬 vs Vercel) 문제가 아니라 클라이언트 UI** 이슈였습니다.
- 지금은 회원가입에서 **년 → 월 → 일 `select`** 로 통일해, 환경과 관계없이 같은 순서로 보입니다.
- API로 보내는 값은 계속 **`YYYY-MM-DD`** (예: `1995-06-15`) 입니다.

---

## 2. “로컬에서 가입했는데 운영에서 로그인이 안 된다”

**거의 항상 DB가 두 개**입니다.

| 확인 | 로컬 `web/.env` | Vercel → Settings → Environment Variables |
|------|-----------------|-------------------------------------------|
| `DATABASE_URL` | Neon 연결 문자열 | **동일한** 문자열이어야 동일 DB |

- 로컬만 Docker Postgres, 운영만 Neon이면 **사용자 테이블이 갈라집니다.**
- **같은 Neon 브랜치 URL**을 양쪽에 넣으면 로컬 가입 ≒ 운영 DB 반영입니다.

`AUTH_URL`은 **로컬**은 `http://localhost:3000`, **운영**은 `https://실제도메인` 처럼 **각자 주소**로 두면 됩니다 (`DATABASE_URL`만 같으면 됨).

## 3. “운영에서 가입/로그인이 이상하다” 체크리스트

### A. 가입 요청이 막히거나 503

1. 브라우저 **F12 → Network** → `register` 제출 직후 **`/api/register`** 클릭  
2. **Status**, **Response** 본문 확인  
3. Vercel → 해당 **Deployment → Logs / Functions** 에서 같은 시각의 에러 확인  

**자주 나오는 원인**

- `DATABASE_URL` 미설정·오타·Preview만 설정됨
- Neon URL 끝의 `&channel_binding=require` → 지우고 `sslmode=require` 위주로 재시도

### B. 가입은 되는데 로그인만 안 됨

- 운영에서는 기본적으로 `registrationApproved` 가 `false` 일 수 있습니다.  
- **Vercel**에 `REGISTRATION_AUTO_APPROVE=1` 넣고 Redeploy 하거나, Neon **SQL Editor**에서 해당 사용자 행의 `registrationApproved` 를 `true` 로 맞춥니다.

Neon에서 최근 가입자 확인 예시 (테이블 이름은 Prisma 기본 `"User"`):

```sql
SELECT email, "registrationApproved", "birthDate", "createdAt"
FROM "User"
ORDER BY "createdAt" DESC
LIMIT 20;
```

### C. 로컬에서 끝까지 재현

1. `web/.env` 의 `DATABASE_URL` 을 **운영과 동일**하게 설정  
2. `npm run dev`  
3. 회원가입 후 위 SQL로 Neon에서 행이 생겼는지 확인  

---

## 4. 마이그레이션만 로컬에서 실패할 때

- Neon **Pooler** URL로는 가끔 `migrate deploy` 가 실패합니다.  
- 그때는 Neon 콘솔의 **Direct(비-pooler)** 연결 문자열로 **로컬에서만**  
  `npx prisma migrate deploy` 실행하면 됩니다.  
- 앱·Vercel 런타임은 Pooler URL을 써도 됩니다.

---

## 5. 요약

| 증상 | 먼저 볼 것 |
|------|------------|
| 달력 순서만 다름 | 브라우저 `type="date"` 특성 → 이제 select 로 통일됨 |
| 로컬 가입 ≠ 운영 로그인 | `DATABASE_URL` 이 다른지 |
| 운영 가입 API 503 | Vercel 환경 변수·Neon URL·Network/Logs |
| 운영 가입 후 로그인 불가 | 승인 플로우·`REGISTRATION_AUTO_APPROVE`·DB 플래그 |
