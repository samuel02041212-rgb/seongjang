# Neon 처음부터 끝까지 — 아무것도 모를 때 따라하기

이 문서는 **컴퓨터를 처음 켠 것처럼** 단계만 적었습니다.  
**MongoDB Atlas** 때처럼 “인터넷上 DB 주소를 받아서 내 프로젝트에 넣는다”고 생각하면 됩니다.  
(이름만 **Neon**이고, DB 종류는 **PostgreSQL**입니다.)

---

## 시작 전에 (30초)

- **인터넷** 연결되어 있어야 합니다.
- **이메일** 또는 **Google / GitHub 계정** 하나 준비해 두세요. (Neon 가입에 씁니다.)

---

## A. Neon 사이트에 들어가서 계정 만들기

### A-1. 사이트 열기

1. **크롬** 또는 **Edge** 브라우저를 켭니다.
2. 맨 위 주소창을 클릭합니다.
3. 아래 주소를 **그대로** 입력하고 **Enter** 를 누릅니다.

   `https://neon.tech`

4. Neon 첫 화면이 열리면 성공입니다.

### A-2. 가입(Sign up)

1. 화면에서 **Sign up** 또는 **Start for free** / **Get started** 같은 **가입** 버튼을 찾아 누릅니다.  
   (한글로 되어 있으면 **가입** / **무료로 시작** 비슷한 말이 나옵니다.)
2. 로그인 방식을 고릅니다. 보통 아래 중 하나입니다.
   - **Google** / **GitHub** / **이메일**
3. 고른 방식대로 안내를 따라갑니다.
   - Google/GitHub면 **허용** 또는 **계속** 을 누르면 됩니다.
   - 이메일이면 **이메일 주소** 넣기 → 메일함에 온 **인증 링크** 클릭(나올 경우)
4. 끝까지 하면 **로그인이 된 상태**로 Neon 안으로 들어갑니다.

> “이미 계정이 있다”고 나오면 **Log in** 으로 로그인만 하면 됩니다.

---

## B. 데이터베이스 프로젝트 만들기 (Neon 안에서 한 번만)

처음 들어오면 **프로젝트 없음** 이라서, **DB 한 묶음(프로젝트)** 을 하나 만들어야 합니다.

### B-1. 프로젝트 생성 버튼

1. 화면에서 **Create project** 또는 **New project** / **프로젝트 만들기** 를 누릅니다.

### B-2. 이름과 지역 (설정)

1. **Project name** (프로젝트 이름): 아무거나 괜찮습니다. 예: `seongjang`
2. **Region** (지역): 목록에서 **가까운 지역**을 고르면 됩니다.  
   (한국에서 많이 쓰는 건 **Asia (Tokyo)** 또는 **Singapore** 정도가 나올 수 있습니다. 없으면 **기본값 그대로** 둬도 됩니다.)
3. **PostgreSQL 버전**은 **건너뛰거나 기본값** 그대로 두어도 됩니다.

### B-3. 만들기

1. **Create project** / **Create** 를 누릅니다.
2. **1분 안팎**으로 로딩이 돌다가, 대시보드(관리 화면)로 넘어가면 성공입니다.

---

## C. 연결 문자열 복사하기 (가장 중요)

이 한 줄이 예전 **Atlas 연결 주소**와 같은 역할입니다.

### C-1. 연결 정보 창 열기

1. 프로젝트가 만들어진 직후, 화면에 **Connection string** / **Connect** / **Connection details** 같은 영역이 **이미 보일 수 있습니다.**
2. 안 보이면:
   - 왼쪽에서 **내 프로젝트** 가 선택돼 있는지 확인하고
   - 상단이나 가운데에 **Connect** / **Connect to your database** 버튼을 눌러 봅니다.

### C-2. 복사할 내용 고르기

1. **Connection string** 또는 **URI** 탭이 있으면 그걸 누릅니다.
2. 목록에서 **Prisma** 또는 **Node.js** 라고 되어 있으면 **그걸 고르면 편합니다.**  
   (없으면 **psql** 말고, **`postgresql://` 로 시작하는 문자열**이 보이는 항목을 고릅니다.)
3. 긴 한 줄이 보입니다. 맨 앞이 **`postgresql://`** 인지 확인합니다.
4. 옆에 **Copy** / **복사** 버튼이 있으면 눌러서 복사합니다.  
   없으면 그 줄을 **전체 드래그해서 복사(Ctrl+C)** 합니다.

### C-3. 비밀번호가 가려져 있을 때

- 문자열 안 비밀번호 자리가 `****` 이거나 비어 있으면 **복사한 줄이 동작하지 않을 수 있습니다.**
- 화면에 **Reveal secret** / **Show password** / **Reset password** 가 있으면 눌러서 **비밀번호가 들어간 완전한 문자열**을 다시 복사합니다.

## D. 내 컴퓨터에 붙여넣기 (`web/.env`)

### D-1. 폴더와 파일

1. **파일 탐색기**를 엽니다.
2. 이 경로로 갑니다: `D:\seongjang\web`
3. **`.env`** 라는 파일이 있는지 봅니다.
   - **있으면**: 메모장이나 Cursor로 엽니다.
   - **없으면**: 같은 폴더에 **`.env.example`** 파일을 **복사**한 뒤, 이름만 **`.env`** 로 바꿉니다. 그다음 엽니다.

### D-2. DATABASE_URL 수정

1. 파일 안에서 **`DATABASE_URL`** 이라고 적힌 줄을 찾습니다.
2. 형태는 이렇습니다.  
   `DATABASE_URL="여기에뭔가"`
3. 따옴표 **`"` 안의 내용 전체**를 지웁니다.
4. 아까 Neon에서 복사한 줄을 **한 번에 붙여넣기** 합니다.  
   (절대로 줄을 나누지 마세요. 한 줄이어야 합니다.)
5. 결과는 대략 이런 모양입니다.

   `DATABASE_URL="postgresql://알아서긴주소...neondb?sslmode=require"`

6. **저장** 합니다 (Ctrl+S).

---

## E. 테이블 만들기 & 첫 데이터 넣기 (명령창)

### E-1. 명령 창 열기

1. 파일 탐색기에서 **`D:\seongjang`** 폴더를 엽니다.
2. 위 **주소창**을 클릭합니다.
3. `cmd` 라고 치고 **Enter** 를 누릅니다.  
   → 검은 창이 열립니다.

### E-2. 명령 세 줄 (순서대로)

**한 줄씩** 복사해서 붙여넣고, 매번 **Enter** 를 누릅니다.

```text
cd web
```

```text
npx prisma migrate deploy
```

- 끝에 **Applied migration** / **성공** 비슷한 말이 있고, 빨간 에러가 없으면 OK입니다.
- **연결 실패**라고 나오면 → Neon에서 복사한 `DATABASE_URL`이 맞는지, 따옴표가 잘 닫혔는지 `.env` 를 다시 봅니다.

다음:

```text
npm run db:seed
```

- **`Seed OK`** 비슷한 문구가 나오면 성공입니다.

개발용 계정(시드로 들어감):

- 이메일: `dev@seongjang.local`
- 비밀번호: `devpassword`

## F. 웹 사이트 켜 보기

같은 검은 창에서 (`web` 폴더에 있는 상태):

```text
npm run dev
```

브라우저에서 주소:

`http://localhost:3000`

## 막혔을 때

| 상황 | 할 일 |
|------|--------|
| Neon 가입 화면이 안 보임 | 주소가 `neon.tech` 맞는지, 링크 복사 붙여넣기 |
| `.env` 파일이 안 보임 | 파일 이름이 정확히 `.env` 인지 (`.env.txt` 아님) |
| migrate 에서 DB 연결 실패 | `DATABASE_URL` 따옴표 안이 Neon에서 복사한 **완전한 한 줄**인지 |
| 비밀번호를 모르겠음 | Neon 프로젝트 → 비밀번호 보기/재설정 |

---

## 왜 Atlas가 아니라 여기인가요? (짧게)

- **Atlas = MongoDB**, 지금 프로젝트 = **PostgreSQL** 이라 서로 다릅니다.
- **하는 일은 같습니다:** 클라우드에 DB 두고, **연결 문자열**을 내 `.env`에 넣어서 개발합니다.
